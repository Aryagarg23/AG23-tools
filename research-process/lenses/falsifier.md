# The Falsifier

> **Stance:** Don't defend your hypothesis — try to execute it.

**Use when** you have a hypothesis you *like*. That is exactly when it's most
dangerous.

## Core questions

- What single observation would prove this **wrong**? Have I gone looking for it?
- What's the most charitable **alternative explanation**, and have I ruled it out?
- Am I seeing the mechanism, or a coincidence that happens to fit?
- What did I **ignore** because it didn't fit the story?

## Method

1. State the **killer observation** up front — the one result that refutes you.
2. Design the probe most likely to **produce** it, not to confirm you.
3. If the stakes are high, spawn independent skeptics each prompted to
   refute; kill the claim on a majority refute.
4. Promote to "finding" only what survives the attempt to kill it.

## Disciplines

- Refute-first: default to "not real" until it survives.
- A confirmation run is nearly worthless; an adversarial run is the price of belief.
- Watch for the **self-comparison trap** — a pipeline that hands you your own
  input back as its output and invites you to call it agreement.

## Hypothesis form

"H is confirmed *only because* probe P — designed to produce refuting
observation O — failed to produce O."

## Signature move

Name the observation that would kill it, then go get that observation.

## Worked example — Adaption

The auto-reflector "confirmed determinism": same grade, same score, no drift.
The Falsifier asks *what would this look like if it were an artifact rather
than a fact?* — and finds the killer observation: the "two runs" shared one
dataset id, so the reflection compared the control **to itself**. Voided. The
durable rule it produced: an auto-reflector narrates whatever the state
machine feeds it — so the state machine must never feed it a self-comparison.
